# Vault of vaults - experiment notebooks

This folder contains a progressive series of Jupyter notebooks exploring allocation strategies for a "vault of vaults" portfolio — a meta-strategy that invests across multiple DeFi yield vaults (primarily on Hyperliquid, Arbitrum, and other chains).

The series evolves from hand-picked vault prototypes (NB01–NB09) through cross-chain diversification (NB10–NB25), systematic parameter search (NB26–NB45), Calmar signal engineering (NB46–NB53), probabilistic allocation methods (NB54–NB65), dual-signal architectures (NB66–NB70), rank composite metadata selection (NB71), and finally cash deployment optimisation with HLP absorption and dispersion-aware weighting (NB72–NB73).

## Reference of indicators

| Indicator | First in | Function name | Description |
|---|---|---|---|
| Rolling Sharpe | NB27 | `rolling_sharpe` | Rolling risk-adjusted return using standard deviation as the risk measure. Standard signal for ranking vault quality over a lookback window. |
| Rolling Sortino | NB26 | `rolling_sortino` | Rolling Sortino ratio penalising only downside deviation. Preferred over Sharpe for asymmetric return distributions typical in DeFi vaults. |
| Rolling Calmar | NB44 | `rolling_calmar` | Rolling return divided by maximum drawdown over a lookback window. Prone to small-denominator amplification when max drawdown is near zero for young vaults. |
| Rolling returns | NB27 | `rolling_returns` | Simple rolling cumulative return over a lookback window. Used as a basic momentum signal for vault ranking. |
| Rolling volatility | NB27 | `rolling_volatility` | Rolling standard deviation of daily returns. Used both as risk signal and as denominator normalisation for Sharpe-based weighting. |
| Calmar EMA | NB50 | `calmar_ema` | Exponential moving average smoothing applied to rolling Calmar ratio. Reduces spike sensitivity compared to raw Calmar. |
| Calmar adaptive EMA | NB50 | `calmar_adaptive_ema` | Volatility-responsive EMA of Calmar: widens smoothing window during high volatility periods. More robust than fixed-window EMA. |
| Calmar exp decay | NB50 | `calmar_exp_decay` | Exponential decay weighting of historical returns before computing Calmar. Recent observations weighted more heavily than older ones. |
| Calmar log sterling | NB50 | `calmar_log_sterling` | Log transformation applied to the Sterling ratio variant of Calmar. Compresses extreme values while preserving signal rank ordering. |
| Calmar confidence weighted | NB50 | `calmar_confidence_weighted` | PSR-inspired sigmoid ramp-up weighting that down-weights young vaults. Prevents extreme Calmar spikes for vaults with short history. |
| Calmar CDaR | NB50 | `calmar_cdar` | Conditional Drawdown at Risk used as the denominator instead of max drawdown. More statistically stable than max drawdown for small samples. |
| Calmar RAMO | NB50 | `calmar_ramo` | Return divided by the average of the maximum drawdowns (RAMO formulation). Alternative denominator reducing single-event sensitivity. |
| Calmar sterling | NB51 | `calmar_sterling` | Sterling ratio variant (denominator = average drawdown with a floor constant). Prevents near-zero denominator amplification. |
| R-squared weighted signal | NB51 | `signal_r_squared_weighted` | Allocates proportionally to the R² of a vault's cumulative return trend. High R² indicates consistent upward drift rather than lucky recovery. |
| UPI (Ulcer Performance Index) | NB48 | `upi` | Ulcer Performance Index: return divided by the root-mean-square of drawdowns. Smoother denominator than max drawdown, reducing spikiness. |
| Pain ratio | NB48 | `pain` | Return divided by the mean absolute drawdown. Less sensitive to single large drawdowns than max-drawdown-based ratios. |
| Sterling floor | NB49 | `calmar_sterling_floor` | Calmar with a constant floor added to the drawdown denominator. Prevents division by near-zero values for vaults with minimal historical drawdown. |
| Log compression | NB49 | `log_sterling` | Log(1 + calmar) transformation. Only method that fully eliminates ceiling-hitting behaviour in young vault signals. |
| Bayesian credibility | NB52 | `bayesian_credibility` | Age-weighted signal using `w_T = T/(T+H)`: young vaults pulled toward group mean; mature vaults express their own signal. Naturally handles short-history vaults without PSR gating. |

| Rolling sterling floor | NB52 | `rolling_sterling_floor` | Rolling Sterling ratio with a constant floor on the drawdown denominator. Combines window-based estimation with small-denominator protection. |
| EMA denoised log sterling | NB52 | `ema_denoised_log_sterling` | EMA smoothing applied to the log-Sterling signal. Reduces noise from day-to-day ratio fluctuations. |
| TVL multiplier | NB70 | `tvl_multiplier` | Adjusts allocation signal by vault TVL. Larger vaults receive a scaling bonus reflecting their liquidity and absorption capacity. |
| Cross-sectional rank | NB49 | `rank` | Ranks vaults by signal value each period and allocates proportionally to rank. Eliminates all small-denominator amplification by design. |
| PSR normalised concentration | NB54 | `make_psr_concentration_function` | Factory function producing continuous concentration limits from PSR. Replaces discrete quality tier caps (Gold/High/Medium) with a smooth PSR-to-cap mapping. |

## Reference of weighting methods

| Method | First in | Description |
|---|---|---|
| Equal weight (1/N) | NB23a | Allocates equally across all selected vaults. Consistently outperforms signal-responsive weighting in NB59–NB65. Simple and robust. |
| Inverse volatility | NB22 | Allocates proportionally to 1/σ. Highest Sharpe ratio (7.15) but lowest CAGR (9.76%). Best for risk-minimisation objectives. |
| Log-signal weighting | NB23b | Allocates proportionally to log-transformed signal values. Compresses extreme signals while preserving rank ordering. |
| Waterfall allocation | NB11 | Greedy cascade: allocate maximum to best-ranked vault, then next, until capital exhausted. Prevents over-diversification; focuses capital on top-conviction vaults. |
| Waterfall diversified | NB28 | Waterfall with per-protocol and per-chain concentration caps to enforce diversification. Balances conviction-based allocation with systemic risk limits. |
| Sortino-based allocation | NB26 | Allocates proportionally to rolling Sortino ratio. Emphasises downside risk-adjusted returns in the weighting. |
| Sharpe-based allocation | NB27 | Allocates proportionally to rolling Sharpe ratio. Standard risk-adjusted weighting signal. |
| Fixed quality tier concentration | NB44 | Sets maximum concentration limits per vault quality tier: Gold=0.33, High=0.20, Medium=0.15, Other=0.10. Simple discrete caps. |
| Concentration multiplier | NB46 | Applies a scalar multiplier to all tier concentration limits, allowing the optimiser to globally tighten or relax position sizing. |
| PSR-based concentration | NB54 | Continuous concentration limit driven by Probabilistic Sharpe Ratio. Replaces discrete tier caps with a smooth PSR-to-concentration mapping. |
| Linear PSR gate | NB59 | Soft gating: PSR below `min_floor` → 0 allocation; PSR in [min_floor, max_ceiling] → linear interpolation; PSR above `max_ceiling` → full allocation. |
| Bayesian credibility weighting | NB60 | Signal value multiplied by `w_T = T/(T+H)`. Young vaults (small T) are shrunk toward the cross-sectional mean; mature vaults express full signal strength. |
| Softmax allocation | NB63 | Allocates proportionally to softmax-transformed signals. Smooth differentiable alternative to rank-based methods; increases concentration in top vaults. |
| Threshold-linear allocation | NB64 | Piecewise linear mapping from signal value to allocation weight with configurable thresholds. Intended as a compromise between equal weight and full signal-proportional sizing. |
| Dual-signal architecture | NB66 | Decouples vault *selection* (gate signal, e.g., Bayesian) from vault *weighting* (e.g., equal weight or rolling Sharpe). Key architectural finding: selection and sizing benefit from different signals. |
| Tiered concentration with HLP | NB72a | Sets concentration tiers by vault maturity/quality, with HLP (Hyperliquidity Provider) as a cash absorption sweep vehicle for undeployed capital. |
| HLP max weight function | NB73 | Dynamic weighting function for HLP cash absorption. Raises HLP's concentration cap dynamically when other vaults are TVL-capped and idle cash accumulates. |
| Dispersion-aware weighting switch | NB72b | Dynamically switches between equal-weight and signal-responsive weighting based on cross-sectional signal dispersion. High dispersion → use signal; low dispersion → use equal weight. In practice the responsive mode activated only 5.9% of periods and underperformed the always-equal-weight baseline (Sharpe 2.93 vs 3.33). |
| HLP cash sweep modes | NB73 | Four strategies for deploying idle cash into HLP: `none` (baseline, no override), `weight_boost` (force HLP to a target portfolio share), `cash_sweep` (redirect TVL-capped overflow to HLP), `two_tier` (reserve a fixed HLP tranche upfront). All override modes underperform the baseline; idle cash functions as a natural drawdown buffer rather than wasted capital. |
